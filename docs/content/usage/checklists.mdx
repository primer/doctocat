## Usage

To understand how checklists work in Doctocat, let's create an accessibility checklist. In this example scenerio, we have a collection of pages documenting UI components. We expect all of the components to meet the following accessibility requirements:

- Typography is legible, even under difficult conditions.
- Focused element is clearly indicated.
- Components use appropriate semantic HTML elements.

We'll use a checklist to make these expectations visible.

### Create a checklist schema

First, we'll need to describe the structure of our checklist by creating a checklist schema. Checklist schema files (`.schema.js`) must live in a `src/@primer/gatsby-theme-doctocat/src/checklists/` directory. In our case, we'll create `a11y.schema.js`:

<!-- NOTE: should schema files be in TypeSript? -->

```js
// @primer/gatsby-theme-doctocat/src/checklists/a11y.schema.js

module.exports = {
  title: 'Accessibility checklist',
  items: {
    legibleType: {
      description: 'Typography is legible, even under difficult conditions.'
    },
    visibleFocus: {
      description: 'Focused element is clearly indicated.'
    },
    semanticHtml: {
      description: 'Component uses appropriate semantic HTML elements.'
    }
  }
}
````

Checklists schema files are JavaScript files that export an object to describe the structure of a checklist. Let's examine the properties of a schema object.

#### `title`

_Type:_ `string`

Give your checklist a title. This becomes the heading of the checklist on documentation pages.

#### `items`

_Type:_ `{ [checklistItemId: string]: { description: string } }` 

Define the checklist items. Each item must have a unique `id` and a `description`.

<Note variant="warning">

Checklist schema files must use CommonJS-style exports (i.e. `module.exports`).

</Note>

### Use a checklist

Now that we've created a checklist schema, we can add our checklist to any page via Markdown frontmatter:



```md
---
a11yChecklist:
  legibleType: true
  visibleFocus: true
  semanticHtml: false
---
```




* The name `a11yChecklist` corresponds to the name of the checklist schema (`a11y.schema.js`).
* The value is an object with a key for each checklist item.
* The value of each key is a boolean indicating whether or not the item should be checked.

<Note variant="warning">
  All items from the checklist schema must be present in the checklist. If a item
  is missing, your site will fail to build. This ensures that you don't
  accidently forget about a checklist item.
</Note>

When checklist frontmatter is present in a Markdown file, the checklist will be rendered on the bottom of the page:
